# demo.md 编写模式

本文件记录了 CAS 项目各模块 demo.md 的编写标准和最佳实践。

### 通用要求

每个子包/core.py 配套的 demo.md 必须满足：

1. **用实参演示，不用形参** — 直接写带真实值的调用示例，不要写抽象的参数表。
2. **多组代码示例** — 输入 + 预期输出（输出在下一行缩进，`→` 前缀）
3. **代码注释** — 每行 Scheme 代码必须带 `;` 注释。Python core.py 带模块 docstring + 逐行注释。
4. **注意事项** — 边界情况、常见陷阱、类型限制
5. **适用场景** — 什么时候用这个函数/类
6. **同类对比** — 与其他方法的差异对照表
7. **最佳实践** — 推荐参数设置和使用模式

**禁止**：把示例放在 Python docstring 中（编辑时意外破坏语法）。

**输出格式**：
```scheme
(call arg1 arg2)
  →  result
```

## 不同模块的 demo 侧重点

### NumPy 数学函数
- 公式/数学含义
- axis 参数说明（0=列, 1=行, None=全部）
- nan-前缀版本的差异
- 数据类型影响

### NumPy FFT (傅里叶变换)
必须包含**实际应用场景**演示。推荐模式：

```
1. 信号生成: 采样率/时间轴/合成信号
2. FFT 分析: 幅度谱/频率轴
3. 频域处理: 滤波/峰值检测
4. 信号恢复: IFFT → 时域
5. 注意事项: 采样定理/频谱泄漏/FFT长度
```

FFT 各函数的demo必须提及其在一维信号处理或二维图像处理中的角色：
- `fft/ifft`: 信号去噪、频谱分析、频域滤波
- `fft2/ifft2`: 图像频域处理（低通模糊、高通边缘）
- `fftshift/ifftshift`: 频谱可视化（零频居中）
- `fftfreq/rfftfreq`: 频率轴标注（分辨率公式 fs/N）
- `rfft`: 实信号频谱分析（音频处理）
- `hfft/ihfft`: 厄米特对称概念（与 rfft 的对应关系）

```scheme
; FFT demo 的标准结构
; 1. 参数设置
(define fs 1000)                   ; 采样率
(define t (np.arange 0 1 (/ 1 fs))) ; 时间轴

; 2. 信号构造
(define sig (np.sin (* 2 pi 50 t)))

; 3. FFT + 频率轴
(define spec (np.fft sig))
(define freqs (np.fftfreq (np.len t) (/ 1 fs)))

; 4. 幅度谱
(define mag (np.abs spec))

; 5. 处理/可视化说明
```

### 机器学习模型
- 参数对结果的影响（C 大小、gamma 大小、n_estimators 等）
- 是否需要特征缩放
- 适合什么类型的数据
- 过拟合/欠拟合的调节方向
- 对比同类模型（如 Lasso vs Ridge, RF vs GBDT）

### 预处理
- 公式（StandardScaler: (x-mean)/std）
- 何时用/何时不用
- 对离群点的敏感性
- 各同类方法的对比表

## 大库拆分到单例级别后的 demo 策略

对 118 个模型、57 个指标、29 个预处理、18 个 FFT 等单例目录：

1. **统一模板**：每个单例 demo.md 使用同一结构 #参数/用法/注意/对比
2. **分类模板**：同类原语共用一段介绍（如"聚合运算"、"三角函数"）
3. **关键函数加料**：用户最常用的函数（np.fft, SVC, RandomForest 等）增加实际应用场景
4. **一般函数规范**：不常用的函数至少包含参数表和基本示例
5. **应用场景优先**：核心函数优先用实际场景演示（信号处理、图像分析、分类任务），而非孤立语法展示
