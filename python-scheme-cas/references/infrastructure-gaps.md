# 基础设施缺失 — 已知限制和待办\n\n## P0 级（阻止主流使用）\n\n### 1. `).method` 点对语法冲突\n\n```scheme\n; 错误 — Scheme reader 将 ) 后的 . 读作点对\n((np.arange 24).reshape 4 6)\n; 变通 — 用中间变量或 . 特殊形式\n(define flat24 (np.arange 24))\n(flat24.reshape 4 6)\n(. (np.arange 24) reshape 4 6)\n```\n\n**原因**: Scheme 点对语法 `.` 前不需要空格。`).` 被 parser 解释为点对。\n**修复方向**: 在 parser 中区分 Context — 只有 `.` 前后都是表达式时才走点对，否则看作标识符。\n\n### 2. 原生切片语法\n\n当前 `arr[2:8:2]` 用 `[arr 2 8 2]` 表达。`2:8:2` 的 `:` 语法无法实现因为 `:` 在 Scheme 中是 keyword 前缀。\n\n**替代方案就绪**: `[arr 2 8 2]` → `arr[2:8:2]`\n\n## P1 级（阻止特定子模块）\n\n### 1. Python for 迭代器\n\nPython `range`, `generator` 等无法直接遍历。有 `py-for-each` 但不够优雅。\n\n```scheme\n; 当前:\n(py-for-each (lambda (x) (display x)) (py-eval \"range(5)\"))\n; 理想:\n(for x in (py-eval \"range(5)\") (display x))\n```\n\n## P2 级（可用但不优雅）\n\n### 1. `+=` 运算符\n\n```scheme\n; Python: prob += expr\n; 当前:\n(prob_lp.__iadd__ (+ xl yl))\n```\n\n### 2. `guard` 异常捕获\n\n`guard` 语法可用但无法正确捕获 py-eval 产生的异常（Python 异常未包装为 Scheme 条件）。\n\n```scheme\n; guard 目前无法捕获 Python 异常\n(guard (ex (else (display \"caught\")))\n  (py-eval \"1/0\"))\n; → 仍然抛出 \"Python eval error: division by zero\"\n```
