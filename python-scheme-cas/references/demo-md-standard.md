# demo.md 编写标准

（2026-06-22 新增，来自 numpy/learn 拆分重构经验）

## 核心原则

demo.md 是**使用文档**，不是 API 参考。目标是让用户看一个 demo.md 就能学会使用这个函数/类。

## 格式规范

### 输出格式

调用代码一行，结果在下一行缩进 + `→`：

```markdown
(np.sum (np.array '(1 2 3 4 5)))
  →  15
```

多行数组结果缩进显示：

```markdown
(np.eye 3)
  →  [[1. 0. 0.]
      [0. 1. 0.]
      [0. 0. 1.]]
```

### 标题

`# 模块/子目录 — 函数名`

### 参数表

| 参数 | 默认 | 类型 | 说明 |
|------|------|------|------|
| `a` | — | 数组 | 输入数组 |
| `:axis` | None | int/None | 操作轴 |

### 对比表

| 指标 | 优势 | 劣势 | 适用 |
|------|------|------|------|
| ... | ... | ... | ... |

## 必须包含的内容

1. **参数说明表** — 所有参数名、默认值、含义
2. **多组代码示例** — 至少 2-3 个不同用法的例子
3. **注意事项** — 边界情况、陷阱
4. **最佳实践** — 推荐的参数设置
5. **同类对比** — 相关的其他函数对比

## 绝对禁止

- ❌ 把示例放在 Python docstring 中（破坏风险高）
- ✅ 始终用独立的 `demo.md` 文件
